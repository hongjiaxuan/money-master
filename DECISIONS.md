# DECISIONS.md 決策紀錄

> 規則：這裡的定案**不重新翻案**，除非用戶主動提出。新的寫在最上面。

## 115/07/02 部署前強制 JSX 編譯把關
- 定案：`deploy.bat` 先跑 `verify_build.js`（Node 離線編譯 index.html 的 JSX），**編譯失敗即中止部署**
- 理由：Babel 瀏覽器即時編譯，單一 JSX 語法錯就整頁白畫面（見 LESSONS）；把關擋在部署前
- 附帶定案：`@babel/standalone` 鎖 `7.26.4`，不升 v8（v8 輸出 ESM 會崩）

## 115/07/02 統計改放財務體檢／年度報表整合進 ReportsView
- 定案：新增「財務體檢」分頁（健康分數）與「年度報表今年 vs 去年同比」，統計主場從 StatsView 移至 ReportsView
- 理由：報表情境集中一頁，HomeView／StatsView 維持精簡
- SW 版本推進至 v5.20

## 115/06/30 情境模擬器放 ReportsView，不放 HomeView
- 定案：What-if 模擬器併入月報 ReportsView
- 理由：HomeView 保持極簡；模擬與報表情境相近
- 否決：HomeView 淨值目標追蹤器、多帳戶現金流規劃（用戶判定實用性不足，已取消）

## 115/06/30 股票只記累積投入成本
- 定案：不記市值、不記損益、不串股價 API；需要時手動累加投入成本
- 理由：市值屬未實現損益，會讓成本基礎淨資產失真，且與常青股利APP職責重疊
- 否決：串接證交所／第三方股價 API

## 115/06/30 範圍鎖定三件事
- 定案：消費記帳＋資產負債＋目標規劃，持倉與股利分析一律排除
- 理由：單一職責，兩個 APP 數字不打架
